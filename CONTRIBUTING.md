# Contributing to NaijaProp

We welcome contributions to the NaijaProp project! This document provides guidelines for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install dependencies: `npm install`
4. Run tests: `npm test`
5. Run contract check: `clarinet check`

## Development Process

1. Create a feature branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Add tests for new functionality
4. Ensure all tests pass
5. Update documentation if needed
6. Commit with descriptive messages
7. Push to your fork
8. Create a Pull Request

## Code Standards

- Follow Clarity best practices
- Add comprehensive tests for new features
- Update README.md for new functionality
- Use clear, descriptive commit messages
- Ensure `clarinet check` passes without errors

## Security

- Never commit private keys or sensitive data
- Report security issues privately to the maintainers
- Follow secure coding practices for smart contracts

## Questions?

Open an issue for questions or discussions about the project.
